# エージェントのRulesとSkillsについて

このドキュメントでは、AIエージェントの挙動をカスタマイズ・拡張するための「Rules（ルール）」と「Skills（スキル）」について説明します。

## 1. Rules (ルール) とは

Rulesは、エージェントが**常に遵守すべき指針や制約**を定義するものです。
プロジェクト全体で統一したいコーディング規約や、コミュニケーションのスタイル、絶対に行ってはいけない禁止事項などを設定します。

### 配置場所と形式
- **場所**: `.agent/rules/` ディレクトリ配下
- **形式**: Markdownファイル (`.md`)
- **読み込み**: エージェントはタスク開始時にこれらの定義を読み込み、自身の「記憶（Memory）」として保持します。

### 内容の例
- **言語設定**: 「常に日本語で回答すること」
- **コーディング規約**: 「Pythonの型ヒントを必須とする」「特定のライブラリの使用を禁止する」
- **振る舞い**: 「ユーザーへの確認を頻繁に行うこと」「重要なファイルの削除前には必ず許可を求めること」

### 作成方法
`.agent/rules/` フォルダ内に新しいMarkdownファイルを作成し、ルールを記述します。
ファイル名はわかりやすいもの（例: `coding_standards.md`, `language_policy.md`）にします。

---

## 2. Skills (スキル) とは

Skillsは、エージェントに**特定の作業手順や専門的な能力**を与えるためのものです。
複雑なセットアップ手順、定期的なメンテナンス作業、特定のAPIを利用した処理など、一連のタスクをパッケージ化してエージェントに教えることができます。

### 配置場所と形式
- **場所**: `.agent/skills/` ディレクトリ配下
- **形式**: 各スキルごとにフォルダを作成し、その中に必須の `SKILL.md` を配置します。
    - 構造例:
        ```text
        .agent/skills/
          └── setup_excel_env/    <-- スキル名フォルダ
                ├── SKILL.md      <-- 手順書（必須）
                └── scripts/      <-- 必要に応じてスクリプトなどを格納
        ```

### 内容の例
- **環境構築**: 「Python仮想環境の作成とライブラリインストール」
- **定型業務**: 「月次レポートの自動生成とメール送信」
- **コード品質チェック**: 「Lintツールの実行と修正提案」

### SKILL.md の構成
`SKILL.md` には、YAML形式のフロントマター（メタデータ）と、Markdown形式の手順記述を含めます。

```markdown
---
name: スキル名
description: このスキルが何をするものかの短い説明
---

# 手順詳細
1. **ステップ1**: ...
2. **ステップ2**: ...
```

エージェントはこの `SKILL.md` を読み込むことで、そのスキル（手順）を実行できるようになります。

---

## 3. まとめ

| 機能 | 役割 | 主な用途 |
| :--- | :--- | :--- |
| **Rules** | **制約・指針** | コーディング規約、言語設定、禁止事項 |
| **Skills** | **能力・手順** | 環境構築、定型タスクの自動化、複合的な処理 |

これらを活用することで、エージェントをあなたのプロジェクトや好みに合わせて最適化することができます。
