# Gitの基本と活用方法

Git（ギット）は、ファイルの「変更履歴」を保存・管理するためのツールです。
プログラムコードだけでなく、ドキュメントの管理にも非常に役立ちます。

## 1. なぜGitを使うのか？

1.  **履歴の保存**: 「いつ」「誰が」「何を」変更したかが全て残ります。「昨日の状態に戻したい」といった場合にすぐに戻せます。
2.  **安全な実験**: 新しいアイデアを試すときに、現在の動いている状態を壊さずに別の場所（ブランチ）で作業できます。
3.  **バックアップと共有**: GitHubなどのクラウドサービスと連携することで、PCが壊れてもデータを失わず、また他人とファイルを共有しやすくなります。

---

## 2. 基本的な概念

- **リポジトリ (Repository)**: ファイルや履歴が保存されている場所（フォルダ）。現在作業中の `excel_work` フォルダもリポジトリです。
- **コミット (Commit)**: ある時点でのファイルの状態を「記録」すること。ゲームの「セーブポイント」のようなものです。
- **ステージング (Staging)**: コミットする前に、どのファイルの変更を記録するかを「選択」するエリア。

---

## 3. よく使う4つのコマンド

作業中に頻繁に使うのは以下の4つです。

### ① 状態を確認する: `git status`
「どのファイルが変更されたか」「どのファイルがまだ記録されていないか」を確認します。
迷ったらまずはこれを打ちましょう。

```bash
git status
```

### ② 変更をステージングする: `git add`
変更されたファイルを、「次のコミットに含めるもの」として選択します。

```bash
# 全ての変更を追加する場合
git add .

# 特定のファイルだけ追加する場合
git add ファイル名
```

### ③ 変更を記録する: `git commit`
ステージングした内容を、メッセージ付きで保存します。メッセージには「何をしたか」を書きます。

```bash
git commit -m "エージェントルールの説明ファイルを追加"
```

### ④ 履歴を見る: `git log`
これまでのコミット履歴を確認します。

```bash
git log
```

---

## 4. 基本的なワークフロー

1.  ファイルを**編集・作成**する。
2.  `git status` で変更を確認する。
3.  `git add .` で変更をステージングエリアに乗せる。
4.  `git commit -m "変更内容の説明"` で記録（セーブ）する。

このサイクルを繰り返すことで、安全かつ確実に成果物を管理できます。

## 5. 次のアクション

このガイドを読んだ後、以下のコマンドを実行して現状を保存することをお勧めします。

```bash
git add .

```
